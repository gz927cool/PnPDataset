# **历史索引原子级语义抽取规范 V5.3 (生产级全链路爆破版)**

本规范定义了将艺术史文献索引转化为原子级、高精度知识图谱三元组的终极工程化流程。

## **一、 算法核心哲学 (The Core Philosophy)**

1. **地毯式爆破 (1:N Explosion)**：  
   * 严禁任何形式的合并。一行原始索引必须根据其蕴含的维度爆破为 N 行原子三元组。  
   * **地理级联爆破**：任何地标实体必须强行拆解为 建筑 \-- is\_located\_in \-- 城市 和 城市 \-- is\_located\_in \-- 国家 两行。  
   * **作品列表爆破**：艺术家项下的每一个缩进作品、每一个胸像订单、每一组湿壁画，必须拥有独立的 is\_author\_of 记录，严禁在 Object 字段中使用复数或逗号。  
2. **双轨索引机制 (Dual-Track Indexing)**：  
   * **序号 (Index)**：记录输出表的绝对行位置（1..568..N），用于核对全量数据规模。  
   * **溯源行号 (Row ID)**：记录该数据在原始索引表中的物理行号，用于 1:N 的审计对账。  
3. **递归深度闭环 (Recursive Loop Resolution)**：  
   * 针对 see under 指令，严禁产生“逻辑断头台”。必须自动触发跨条目检索，将被引用目标的身份、地位、核心资助行为直接回填至当前 Row ID。  
4. **零匿名占位 (Zero Anonymity)**：  
   * 严禁使用 Portrait X 或 Bust 1。遵循 **SENP 5.0 协议**，利用语境和外部验证构造唯一性描述名称。  
5. **存续状态强标识 (Status Mandatory)**：  
   * 每一个创作类三元组必须标注：Confirmed (确有其作), Planned (史实计划/未动工), Lost (已毁), Genre (题材特长)。

## **二、 语义处理六大算子 (The Processing Operators)**

### **1\. 触发式原文检索算子 (TCR)**

* **指令**：定位页码对应的 CHP-\*.md **正文与脚注**。  
* **任务**：将索引中的代词（如 "The Cardinal"）还原为全名（如 "Francesco Barberini"）。

### **2\. 互联网跨向验证算子 (WACV)**

* **工具**：Getty ULAN/TGN/AAT, Wikipedia, RCT (英国皇室收藏) 目录。  
* **任务**：核实生卒年，标准化地理层级，确认“未动工”委任。

### **3\. 地理扁平化算子 (Geo-Flattening)**

* **逻辑**：任何涉及物理空间落位的对象，必须紧随一行该空间的行政隶属关系三元组。  
* **示例**：(Bernini) \-- is\_author\_of \-- (Baldacchino) $\\rightarrow$ (Baldacchino) \-- is\_housed\_at \-- (St Peter's) $\\rightarrow$ (St Peter's) \-- is\_located\_in \-- (Vatican City)。

### **4\. 肖像多维映射算子 (Portrait Mapping)**

* **逻辑**：肖像记录必须拆解为：(作者) \-- is\_author\_of \-- (作品) \+ (作品) \-- depicts \-- (对象) \+ (作者) \-- has\_social\_link \-- (对象)。

### **5\. 中介代理链解构算子 (Broker Chain)**

* **逻辑**：将 A 通过代理 B 向 C 购买作品 D 拆解为：(B) \-- agent\_for \-- (A) \+ (A) \-- commissioned\_via \-- (B) \+ (D) \-- acquired\_by \-- (A)。

### **6\. 题材隔离算子 (Genre Isolation)**

* **逻辑**：将泛指的题材（Landscapes, Animals）标记为 Genre 状态，并使用 specialized\_in 谓词。

## **三、 核心谓词逻辑库 (标准语义边界)**

| 分类 | 精细化谓词 (Predicate) | 语义边界说明 |
| :---- | :---- | :---- |
| **社会/身份** | **spouse\_of** | 仅限正式配偶。 |
|  | **agent\_for** | 艺术、财务或商业代理。 |
|  | **rival\_of** | 明确记录的竞争与对抗。 |
|  | **protector\_of** | 政治或经济庇护。 |
| **创作/存续** | **is\_author\_of** | 确证存在的作品（Status: Confirmed）。 |
|  | **planned\_to\_create** | 有委任但证实未动工的项目（Status: Planned）。 |
|  | **depicts** | 作品表现的视觉主体。 |
|  | **specialized\_in** | 艺术家的流派专长。 |
| **空间/地理** | **is\_housed\_at** | 物理现存地（不可移动/馆藏）。 |
|  | **is\_located\_in** | 行政层级包含关系。 |
|  | **settled\_in / active\_in** | 定居地或核心职业活跃地。 |

## **四、 输出格式与审计标准**

### **1\. 结果表格定义 (Markdown/CSV)**

* **序号 (Index)**：解析产出的连续编号。  
* **主体 (Subject)**：原子化实体名。  
* **谓词 (Predicate)**：标准谓词。  
* **客体 (Object)**：标准化原子实体（SENP）。  
* **状态 (Status)**：Confirmed / Planned / Lost / Genre。  
* **深度核验逻辑**：TCR/WACV 的证据链。  
* **Row ID**：原始索引的锚点行号。

### **2\. 二次完整性审计逻辑**

* **规则 1**：原始数据中的所有行（基于数据集实际行数），每一行 ID 必须在输出结果中出现至少一次。  
* **规则 2**：检查爆破率。若作品列表或地理层级未产生多行输出，则判定为“解析不全”。

## **五、 状态定义补充**

* **Confirmed**：经多源证实的作品或行为。  
* **Planned**：史实证明的委任意愿，但由于英国内战、艺术家拒绝、财务中断等原因未产生实物。  
* **Genre**：指代流派倾向（如“擅长画海景”），非特定单体。
