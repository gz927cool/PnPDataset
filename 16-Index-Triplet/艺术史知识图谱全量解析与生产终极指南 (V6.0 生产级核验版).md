# **艺术史知识图谱全量解析与生产终极指南 (V6.0 生产级核验版)**

本指南确立了将艺术史叙事文献（如 Francis Haskell 的著作）转化为高精度、原子级、全生命周期可溯源知识图谱的终极工程标准。

## **一、 核心算法哲学 (The Core Engineering Logic)**

### **1\. 1:N 绝对原子化地毯式爆破 (Absolute Atomicity)**

* **原则**：严禁合并任何语义维度。每一个三元组必须是不可再分的最小逻辑单元。  
* **纯净化命名 (Purified Naming)**：Subject（主体）与 Object（客体）列**仅允许出现实体的正式名称**。  
  * **禁令**：严禁在名称中包含状态（Confirmed\_）、材质（Marble\_）、类型（Oil\_）或作者（by\_X）等描述性前缀。  
* **属性剥离（二次爆破算子）**：作品的材质、年代、类型、作者必须从原始描述中彻底剥离，并转化为独立的三元组行。  
  * 例：Raphael, Deposition, 1507 必须爆破为：  
    1. (Raphael) \-- is\_author\_of \-- (Deposition)  
    2. (Deposition) \-- created\_in \-- (1507)  
    3. (Deposition) \-- has\_material \-- (Oil on Wood) (需通过联网补全)

### **2\. 地理全链路级联 (Geographic Cascade)**

* 任何地理位置实体必须溯源至国家级，严禁出现“地理孤岛”。  
* **级联路径**：地标/建筑 \-\> 位于 \-\> 城市；城市 \-\> 位于 \-\> 省份(如有) \-\> 位于 \-\> 国家。

### **3\. See Under 深度溯源机制 (Deep Context Retrieval)**

* **触发动作**：遇到索引中的 see under \[目标\] 指令时，严禁仅在索引表内打转。  
* **回溯机制**：必须利用目标项关联的页码，**实时回溯检索 NotebookLM 中的原书 MD 文档（如各 CHP-X.md 章节）**，阅读具体文字段落。  
* **回填生成**：提取主体在该语境下的资助、代理或社交细节，将“空洞”的跳转转化为具备实质内容的原子三元组。

## **二、 字段定义标准 (Standard Data Schema)**

| 字段名 | 强制规范与定义 |
| :---- | :---- |
| **序号 (Index)** | 输出表的全局连续物理行号，用于统计总规模 T。 |
| **溯源行号 (Row ID)** | 对应原始索引 MD 文件的物理行号，用于 1:N 审计对账。 |
| **主体 (Subject)** | 经过去匿名化补全后的标准全名（如：Smith, Joseph 而非 Consul Smith）。 |
| **谓词 (Predicate)** | 必须使用标准谓词库（如 agent\_for, created\_in, depicts, acquired\_by 等）。 |
| **客体 (Object)** | **核心：仅允许正式名称。** 严禁带有 Status、属性前缀或描述性后缀。 |
| **状态 (Status)** | 枚举：Confirmed, Planned, Lost, Destroyed\_by\_Fire, Genre, Misattributed。 |
| **依据 (Source)** | 标注具体页码（Page X）或生成逻辑（Cascade, Grounding, Recursive）。 |
| **核验逻辑 (Evidence\_Logic)** | **必填：描述该行生成的推导证据链。** 格式：\[算法标签\]: 逻辑描述。 |

## **三、 全量触发机制 (Trigger Mechanisms)**

### **1\. 外部联网强制触发 (Grounding Triggers)**

* **\[职衔去匿名化\]**：出现 Consul, Duke, Cardinal 等代称时，强制查证全名及任职时段。  
* **\[属性缺失补全\]**：在剥离属性时，若作品材质、创作精确年份、现藏地不明，强制启动外部查询。  
* **\[存续确认\]**：原文提及“formerly in”或“dispersed”时，核实作品是否被毁（Destroyed\_by\_Fire）或下落不明（Lost）。

### **2\. 结构爆破触发 (Explosion Triggers)**

* **\[属性复合\]**：一行描述包含多个维度（如：“大理石胸像，1641年”）时，触发多行爆破。  
* **\[中介/资助链\]**：提到“通过某人购买”时，拆解为代理关系（agent\_for）与委托路径（commissioned\_via）。

### **3\. 深度跳转触发 (Deep Retrieval Triggers)**

* **\[See Under 指令\]**：根据关联页码跳转至 NotebookLM 中的章节原文，抓取实质性语义。

## **四、 核验逻辑 (Evidence\_Logic) 标签体系**

解释列必须使用以下标准化标签开头，以确保审计可复核：

* **\[二次爆破\]**：说明如何将年份、材质、作者从作品名中彻底剥离并生成新行。  
* **\[地理级联\]**：说明从微观地标到国家的级联证据（如：TGN 校验过程）。  
* **\[递归跳转\]**：**\[核心\]** 详细记录回溯了 NotebookLM 哪个章节 MD 及哪一页，抓取了什么具体语境。  
* **\[身份固化\]**：说明如何锁定匿名职衔（如：Wiki 确认其为英国领事 Joseph Smith）。  
* **\[SENP 剥离\]**：说明为何移除原有描述性前缀（如 Confirmed\_）以实现纯净化命名。  
* **\[WACV\]**：(Within-Article Context Verification) 基于书中上下文的史实推断。

## **五、 审计与审查规范 (Audit Report)**

每份产出文档必须在解析结果表之前包含以下模块：

### **1\. 数据完整性审计报告**

* **物理行数**：原始索引扫描范围（Row ID X \- Row ID Y）。  
* **有效三元组数**：爆破生成的总记录数 T。  
* **爆破率 (T/R)**：要求 \> 2.5（若作品多，爆破率常达 4.0 以上）。  
* **审计状态**：**PASS**（需 100% 满足去匿名、级联、属性剥离与原文回溯）或 **FAIL**。

### **2\. 差异分析 (Gap Analysis)**

记录未产出三元组的 Row ID 及其原因：

* \[Header\]：分类标题。  
* \[Recursive Resolved\]：语义已通过跳转机制回填至其他关联行。  
* \[Repeating\]：重复索引项。

## **六、 结果文档命名规则 (Naming Protocol)**

**强制格式**：Haskell\_\[Letter\]\_Atomic\_V6.0\_R\[Start-End\]\_T\[Total\]\_\[YYYYMMDD\].md

* **Letter**：处理的索引首字母（如 Q-R）。  
* **R\[Start-End\]**：原始 MD 索引文件中的物理行号范围。  
* **T\[Total\]**：该文件产出的原子三元组最终总物理行数。  
* **YYYYMMDD**：生成日期。

## **七、 严禁事项 (Forbidden List)**

1. **严禁**在 Object 列使用任何下划线前缀（如 Confirmed\_, Oil\_ 等描述词）。  
2. **严禁**在客体名称中合并任何属性信息（必须二次爆破）。  
3. **严禁**在最终结果中保留跳转指令（必须通过原文回溯转化为事实三元组）。  
4. **严禁**出现地理孤岛（所有地名必须链接至国家级）。
