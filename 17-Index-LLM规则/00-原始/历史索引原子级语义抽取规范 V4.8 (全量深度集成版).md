# **历史索引原子级语义抽取规范 V4.8 (全量深度集成版)**

本规范是艺术史知识图谱构建的终极技术标准。它集成了从 V4.1 到 V4.7 的所有逻辑，并针对复杂历史语境下的“中介代理”、“时序矛盾”及“存续状态”进行了深度补强。

## **一、 算法核心哲学 (The Core Philosophy)**

1. **绝对原子化 (Absolute Atomicity)**：每一个三元组必须是不可再分的最小逻辑单元。  
2. **关系具象化 (Concrete Relations)**：严禁使用模糊谓词。必须通过 TCR/WACV 还原为具体的社会/经济属性（如 agent\_for, creditor\_of）。  
3. **时序闭环校验 (Temporal Closure)**：**\[V4.8 强化\]** 强制验证 (Subject) 与 (Object) 的存续时间交集。若创作时间晚于艺术家卒年，必须自动修正谓词为 attributed\_to 或标记为 Status: Error。  
4. **存续与真实性标识 (Status Flag)**：每一条涉及作品的三元组必须强制标注：Confirmed (确有其作), Planned (计划/未执行), Lost (已毁), Attributed (传为某人), Replica (副本)。  
5. **题材与实体隔离 (Genre vs. Entity)**：区分艺术家的“擅长题材（Genre）”与“具体作品（Specific Work）”。  
6. **场域逻辑全路径 (Venue Provenance)**：记录作品的 original\_location (原位), current\_location (现藏) 及 transit\_point (流转点)。

## **二、 语义处理流 (Data Pipeline)**

### **1\. 过滤与初筛站 (The Filters)**

* **动态语义提取**：识别索引中蕴含的行为动词。若仅有页码，标记为 \[已过滤: 仅统计占位\]。

### **2\. 内部触发式原文检索站 (TCR Station)**

* **深度语境还原**：根据页码锚定 CHP-\*.md。  
* **特定任务**：识别指代对象（如“他”指代哪位伯爵）、捕捉隐藏的第三方中介（Broker）。

### **3\. 互联网跨向验证站 (WACV Station) —— \[V4.8 专项指导\]**

* **Wikipedia 任务**：锁定生卒年、主要活动城市、重大历史事件。  
* **Getty ULAN/AAT 任务**：  
  * **ULAN**：规范化人名及其多国语言别名。  
  * **AAT**：对客体进行分类（如将“Bust”细化为 Sculpture/Portrait Bust，将“Landscapes”定性为 Genre）。  
* **Google (AI 模式/学术搜索) 纠偏**：  
  * 针对争议性委任（如 Bernini-Henrietta 案）寻找最新的艺术史研究成果。  
  * 验证作品是否曾毁于大火、战争或是否从未动工。  
* **冲突处理逻辑**：若 WACV 发现原著索引暗示的关系在史实上是不成立的（如年代错误），必须在三元组中体现 planned\_to 或 misattributed，并保留页码作为来源参考。

## **三、 核心谓词逻辑库 (V4.8 扩展标准)**

### **1\. 社会、婚姻与代理 (去模糊化)**

* **spouse\_of**：配偶。  
* **agent\_for** / **broker\_for**：商业、财务、艺术代理人。  
* **rival\_of** / **political\_opponent\_of**：社交冲突。  
* **pupil\_of** / **teacher\_of**：师徒关系。

### **2\. 创作、流派与存续**

* **is\_author\_of**：确认完成的作品。  
* **planned\_to\_create** / **commissioned\_but\_unexecuted**：未完成的计划。  
* **specialized\_in**：艺术家的擅长题材 (Genre)。  
* **depicts**：作品描绘的对象。  
* **replica\_of**：作为……的副本。

### **3\. 空间与物理轨迹 (Venue)**

* **born\_in** / **died\_in**：生命端点。  
* **settled\_in**：长期定居。  
* **is\_housed\_at**：现存地点。  
* **originally\_created\_for**：原始安置场域。  
* **removed\_from**：从某地迁出。

## **四、 高级推理算子 (Advanced Reasoning)**

### **1\. 中介代理链算子 (Intermediary Chain)**

* **逻辑**：将 A 委托 B 找 C 创作 D 拆解为：  
  1. (A) \-- commissioned\_via \-- (B)  
  2. (B) \-- agent\_for \-- (A)  
  3. (B) \-- commissioned \-- (C)  
  4. (C) \-- is\_author\_of \-- (D)

### **2\. 肖像计划溯源算子 (Planned Work Operator)**

* **逻辑**：若验证发现作品未完成，自动拆解为：  
  1. (艺术家) \-- planned\_to\_create \-- (作品)  
  2. (赞助人) \-- intended\_to\_commission \-- (艺术家)  
  3. (参考物) \-- used\_as\_model\_for \-- (计划作品)

### **3\. 地理爆破与包含算子**

* **逻辑**：(地点) \-- is\_part\_of \-- (更大行政区) 的层级化。

## **五、 元数据与状态标签表 (Metadata Flags)**

| 标签 (Flag) | 含义 (Meaning) |
| :---- | :---- |
| **Confirmed** | TCR 与 WACV 双向证实。 |
| **Planned** | 史料证实有委任意愿，但无实物证据或已证实未动工。 |
| **Lost** | 历史上确凿存在，但已毁或失踪（需标注损毁原因，如 Fire, War）。 |
| **Genre** | 泛指流派或题材，非特定单体。 |
| **Attributed** | 存疑，部分专家认为系某人所作，而非确证。 |
