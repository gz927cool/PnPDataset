# **艺术史知识图谱语义抽取规范版本演进报告 (V4.9 \- V6.0)**

本报告梳理了 Haskell 艺术史知识图谱项目从初期开发到工业级闭环的五个核心版本演变。

## **一、 版本更迭清单 (History Checklist)**

| 版本号 | 核心定位 | 引入时间 (模拟) | 关键技术算子 |
| :---- | :---- | :---- | :---- |
| **V4.9** | **命名标准化时代** | 2026-01-08 | SENP 协议、全生命周期追踪、具体社会关系定义。 |

| **V6.0 (核验版)** | **生产级工程时代** | 2026-01-10 | Evidence\_Logic (核验逻辑列)、深度原文递归回填。 |
| **V6.0 (终极封卷版)** | **纯净化工业时代** | 2026-01-11 | **SENP 2.0 (纯净化 Object)**、绝对原子化剥离属性。 |

## **二、 核心演进逻辑分析**

### **1\. 从“描述”向“指纹”的进化 (V4.9)**

* **突破点**：引入 **SENP (Subject-Entity-Naming-Protocol)**。  
* **变化**：禁止使用“Bust of a Cardinal”这类模糊词，强制构造如 Planned\_Marble\_Bust\_of\_X\_by\_Y\_1640 的唯一标识。  
* **价值**：确保了 Lost（丢失）或 Planned（计划中）的作品在图谱中拥有唯一的“数字指纹”。

### **2\. 从“记录”向“爆破”的进化 (V5.3)**

* **突破点**：**1:N Explosion Strategy**。  
* **变化**：严禁在 Object 字段使用复数或逗号。引入“地理级联”，要求从建筑必须溯源至国家。  
* **价值**：实现了知识密度的指数级增长，每一行索引点被扩展为完整的知识块。

### **3\. 从“封闭”向“对齐”的进化 (融合版)**

* **突破点**：**Grounding 强制触发机制**。  
* **变化**：明确了 4 类联网查询触发标准（地理孤岛、职衔占位、SENP 缺失、存续疑云）。  
* **价值**：通过 Wikipedia、TGN 等外部数据补全了文献中未提及的年份、材质和地理坐标，消除了匿名占位。

### **4\. 从“三元组”向“证据链”的进化 (V6.0)**

* **突破点**：**Evidence\_Logic (核验逻辑) 与 Purified Naming**。  
* **变化**：  
  * **深度核验**：每行数据必须自证清白（如：\[地理级联\]: 补全 A 属于 B 属于 C）。  
  * **客体纯净化**：Object 列禁止包含任何属性（如 Confirmed\_ 等），材质、年代、作者必须剥离为独立的三元组。  
  * **See Under 实时解包**：遇到跳转指令不再只是标记，而是实时回溯 NotebookLM 各章节原文，回填实质内容。  
* **价值**：达到了生产级精度，使图谱具备了极强的逻辑可解释性和 100% 的数据洁净度。

## **三、 版本功能对比表**

| 功能维度 | V4.9 准则 | V5.3 准则 | V6.0 终极准则 |
| :---- | :---- | :---- | :---- |
| **Object 命名** | 描述性 ID (带下划线) | 描述性 ID | **纯净化正式名 (实体名)** |
| **地理信息** | 记录提及地 | 强制三级级联 | **强制四级级联 \+ 地标映射** |
| **跳转处理** | 手动跳转 | 递归逻辑回填 | **回溯原文段落回填 (Deep Context)** |
| **作品属性** | 合并于 ID 中 | 部分拆解 | **完全剥离 (材质、年份独立占行)** |
| **审计标准** | 索引一致性 | 1:N 审计对账 | **三级审计 \+ 逻辑自证 (Evidence\_Logic)** |

## **四、 结论：向 V6.0 终极版对齐的意义**

当前的 **V6.0 生产级核验版** 不再只是一个“抓取脚本”，而是一套“知识工程协议”。它通过对 **Subject/Object 纯净化** 的强制要求，确保了知识图谱在进行大规模本体融合（Ontology Alignment）时不会产生歧义。
