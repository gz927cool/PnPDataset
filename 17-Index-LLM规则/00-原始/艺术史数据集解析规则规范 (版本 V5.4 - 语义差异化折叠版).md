# **艺术史数据集解析规则规范 (版本 V5.4 \- 语义差异化折叠版)**

## **一、 核心逻辑：语义差异化 (Semantic Differentiation)**

V5.4 版本的核心在于不再盲目对每一个页码进行物理拆分，而是根据页码背后的**语义性质**决定处理方式：

### **1\. 语义折叠 (Folding Logic)**

* **适用范围**：泛向记录，包括 active\_in (地理活跃) 和 subject\_of (一般性风格提及、文献记录、无具体动作的提及)。  
* **操作方法**：  
  * 针对同一主体在同一地点的多次活跃记录，或针对同一主体的泛向讨论，**不再生成多行**。  
  * 将其合并为一条记录，并将所有涉及的页码统一填入“深度核验逻辑/备注”列中。  
  * **目的**：显著降低数据冗余，防止高频词（如 Rome, Venice）稀释知识图谱的含金量。

### **2\. 语义爆破 (Exploding Logic)**

* **适用范围**：实质性艺术动作与社会契约。  
* **操作方法**：以下项必须维持 **1:1 爆破**，每一项独立成行：  
  * **具体作品 (Work)**：每一幅画作、雕塑、建筑或书籍。  
  * **赞助契约 (Patronage)**：明确的雇佣、支付或委任关系。  
  * **具体社交/学术链接 (Social/Scholarly Link)**：与其他艺术家、学者或赞助人的直接互动。  
  * **性格/审美特征 (Trait/Taste)**：对主体个性、政策倾向或特定审美偏好的描述。  
  * **经济行为 (Economic Action)**：具体的作品定价、拍卖或交易记录。

## **二、 实体消解与去匿名化 (Entity Resolution)**

* **职衔转化**：使用 has\_identity 谓词，将所有职衔（Cardinal, Pope, Doge, Duke）映射至其全名实体。  
  * *示例*：Pope Innocent X | has\_identity | Gianbattista Pamfili  
* **别名对齐**：将艺术史通称（如 'Monsù Desiderio'）指向其实际身份（如 François Nomé）。  
* **地理补全**：每一个场域节点必须推断其上一级地理路径。  
  * *示例*：Vicenza | is\_located\_in | Italy

## **三、 数据输出结构标准**

| 字段名称 | 解析要求 |
| :---- | :---- |
| **序号 (No.)** | 原子三元组的唯一递增编号。 |
| **主体 (Subject)** | 英语。标准化的实体名称（含全名补全）。 |
| **谓词 (Predicate)** | 英语。使用预设标准谓词（见第四章）。 |
| **客体 (Object)** | 英语。目标实体、属性值或作品名。 |
| **状态 (Status)** | 中文。如“已确认” (Confirmed) 或“已消解” (Resolved)。 |
| **深度核验逻辑** | 中文。包含合并的页码备注、级联推断依据或作品背景说明。 |
| **Row ID** | 原始物理行号。必须与原始 Markdown 文档的行序严格对应。 |

## **四、 V5.4 标准谓词表**

* has\_identity: 用于身份定义、职衔消解及全名映射。  
* active\_in: 地理活跃记录（执行折叠规则）。  
* is\_author\_of: 艺术作品、著作的创作关系。  
* patron\_of / is\_patronized\_by: 赞助、雇佣与委任关系。  
* has\_social\_link: 社交网络、学术通信或师承关系。  
* is\_located\_in: 建筑、宫殿与城市、国家的归属关系。  
* subject\_of: 泛向的讨论、风格评价、文献记录。  
* has\_trait: 性格特征、政策取向或审美偏好描述。

## **五、 审计标准**

1. **逻辑对账**：Row ID 必须连续且可追溯，禁止跳过占位行或重定向行。  
2. **纯净性**：单次解析输出必须严格限定在当前文档范围内，严禁跨文档内容污染。  
3. **密度检查**：检查是否存在重复的、无意义的 active\_in 行。若存在，应强制执行折叠。
