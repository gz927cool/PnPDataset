# **历史索引原子级语义抽取规范 V5.0 (全量全局集成版)**

本规范是针对艺术史著作（以弗朗西斯·哈斯克尔《赞助人与画家》为核心）进行知识图谱构建的终极技术标准。规范涵盖了从非结构化索引到原子级、实证化三元组的全生命周期路径。

## **一、 算法核心哲学 (The Core Philosophy)**

1. **绝对原子化 (Absolute Atomicity)**：  
   * 三元组必须是不可再分的最小逻辑单元。  
   * **地理原子化**：严禁复合地址。Church A, Rome 必须拆分为 (Church A) \-- is\_located\_in \-- (Rome)。  
   * **命名原子化**：Object 字段仅保留实体正式名称，严禁混入状态、作者或年代等冗余属性（见 SENP 协议）。  
2. **语义具象还原 (Restoration)**：  
   * 禁止使用 and, link, relations with 等模糊谓词。  
   * 必须还原为具象动作：is\_patron\_of, spouse\_of, agent\_for, rival\_of。  
3. **存续真实性校验 (Status Verification)**：  
   * 每一个涉及“创作”的三元组必须带有明确的状态标识，区分“史实存在”与“历史愿望/计划”。  
4. **时空一致性守护 (Spatio-Temporal Guardrails)**：  
   * 强制执行生卒年核验，杜绝“死后资助”等逻辑漏洞。  
5. **零匿名占位原则 (No Anonymous Placeholders)**：  
   * 严禁出现 Portrait X 或 Work 1。无法定名的实体必须使用“唯一描述性命名”。

## **二、 语义处理全路径 (The Data Pipeline)**

### **第一步：过滤与初筛 (Filters)**

* **语义判定**：识别索引行是否有动态行为。若仅有页码且无法通过 TCR 补全动作，标记为 \[已过滤: 无动态语义\]。  
* **背景隔离**：排除宏观时代背景描述（如 17th century art）。

### **第二步：内部触发式原文检索 (TCR \- Triggered Contextual Retrieval)**

* **触发条件**：索引项指代模糊（如 "The Cardinal"）、信息残缺、或存在中介暗示。  
* **操作方法**：根据页码定位 CHP-\*.md **正文及脚注 (Footnotes)**。  
* **任务**：消解代词指代，捕捉隐藏的第三方中介（Broker/Agent），识别赞助的真实性质。

### **第三步：互联网/权威库跨向验证 (WACV \- Web-Augmented Cross-Validation)**

* **Wikipedia**：核实生卒年、国籍、核心政治地位。  
* **Getty (ULAN/TGN/AAT)**：  
  * **ULAN**：标准化人物全名（废除绰号，使用正式姓名）。  
  * **TGN**：标准化地理层级。  
  * **AAT**：定性艺术品题材（Genre vs. Specific Work）。  
* **Google AI/Scholar & Museum Catalogs (RCT/National Gallery)**：  
  * **纠偏任务**：针对争议性委任（如 Bernini-Henrietta Maria 案）寻找最新学术结论。  
  * **物理追踪**：锁定 Confirmed 状态作品的现代馆藏编号及存放地。

### **第四步：SENP 5.0 唯一性命名协议 (Strict Entity Naming Protocol)**

* **正式名称优先**：优先使用艺术史公认标题（如 *The Triumph of the Name of Jesus*）。  
* **无名实体构造格式**：若无官方标题，按 \[类型\]\_\[描绘对象\]\_\[作者\]\_\[年代/语境\] 组合，但**输出至三元组 Object 字段时需剥离 ID 标签**，确保原子性。  
* **严禁前缀**：禁止在 Object 字段内添加 Confirmed\_ 或 Planned\_ 字样（此类信息移至 Status 字段）。

### **第五步：高级推理算子 (Advanced Operators)**

1. **中介代理链算子**：将复合交易拆解为 (主顾) \-- commissioned\_via \-- (代理) 与 (代理) \-- agent\_for \-- (主顾) 等子项。  
2. **肖像计划溯源算子**：  
   * 若作品未动工，建立 (艺术家) \-- planned\_to\_create \-- (作品)。  
   * 记录参考物：(参考画作) \-- used\_as\_model\_for \-- (计划雕塑)。  
3. **题材/流派隔离算子**：将泛指的题材（如 Landscapes）通过 specialized\_in 谓词从具体单件作品中剥离。

## **三、 核心谓词逻辑库 (精细化标准)**

| 分类 | 精细化谓词 (Predicate) | 适用场景 |
| :---- | :---- | :---- |
| **社会/经济** | **spouse\_of** | 正式婚姻/配偶关系。 |
|  | **agent\_for** | 财务、商业或艺术采购代理。 |
|  | **rival\_of** | 明确的职业或政治竞争。 |
|  | **pupil\_of / teacher\_of** | 师承关系。 |
| **创作/存续** | **is\_author\_of** | 已确认完成的作品。 |
|  | **planned\_to\_create** | 历史上由于战争、拒绝等原因未执行的计划。 |
|  | **depicts** | 作品所表现的人物或事件。 |
|  | **specialized\_in** | 擅长的流派（Genre/Style）。 |
| **地理/场域** | **is\_housed\_at** | 作品当前的物理存放地（不可移动或馆藏）。 |
|  | **is\_located\_in** | 地理包含关系（如 建筑 在 城市）。 |
|  | **settled\_in / active\_in** | 人物的长期定居地或职业活跃地。 |

## **四、 状态标签与数据元规范 (Metadata & State)**

* **Confirmed**：经 TCR 和 WACV 双向证实的实物或事件。  
* **Planned/Unexecuted**：有明确委任记录但未产生实体的历史计划。  
* **Lost/Destroyed**：历史上曾存在但现已损毁（如毁于火灾）。  
* **Genre**：指代流派倾向而非单件作品。

## **五、 输出格式规范 (Output Format)**

1. **Markdown Table**：用于直观展示推理逻辑、证据链与状态。  
2. **Excel/CSV 标准**：  
   * 字段：Subject, Predicate, Object, Status, Evidence\_Logic。  
   * 编码：UTF-8。  
   * 包含双引号包裹：对含有逗号或特殊符号的字段（如 Evidence）必须使用双引号。

## **六、 典型解析示例 (V5.0 闭环演练)**

原始索引：James II | patron of Gennari | 196  
TCR/WACV 介入：锁定 James II 在圣日耳曼流亡，Gennari 为其随行画家。  
V5.0 原子输出：

1. (James II of England) \-- is\_resident\_of \-- (Saint-Germain-en-Laye)  
2. (Saint-Germain-en-Laye) \-- is\_located\_in \-- (France)  
3. (James II of England) \-- is\_patron\_of \-- (Benedetto Gennari)  
4. (James II of England) \-- Status \-- (Confirmed)

原始索引：Bernini | bust of Henrietta Maria | 179  
纠偏介入：WACV 证实因战争从未动工。  
V5.0 原子输出：

1. (Bernini) \-- planned\_to\_create \-- (Portrait Bust of Henrietta Maria)  
2. (Portrait Bust of Henrietta Maria) \-- Status \-- (Planned)  
3. (Anthony van Dyck) \-- is\_author\_of \-- (Triple Portrait of Henrietta Maria)  
4. (Triple Portrait of Henrietta Maria) \-- used\_as\_model\_for \-- (Portrait Bust of Henrietta Maria)
