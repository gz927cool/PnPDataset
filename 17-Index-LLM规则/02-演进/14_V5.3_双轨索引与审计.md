# **历史索引原子级语义抽取规范 V5.3 (生产级全链路爆破版)**

本规范定义了将艺术史文献索引（如弗朗西斯·哈斯克尔《赞助人与画家》索引表）转化为原子级、高精度知识图谱三元组的终极工程化流程。

## **一、 算法核心哲学 (The Core Philosophy)**

1. **地毯式爆破 (1:N Explosion)**：  
   * 严禁任何形式的语义合并。一行原始索引必须根据其蕴含的维度爆破为 N 行原子三元组。  
   * **地理级联爆破**：任何地标实体必须强行拆解为 建筑 \-- is\_located\_in \-- 城市 和 城市 \-- is\_located\_in \-- 国家。  
   * **作品列表爆破**：艺术家项下的每一个缩进作品、每一个胸像订单、每一组湿壁画，必须拥有独立的 is\_author\_of 记录，严禁在 Object 字段中使用复数或逗号。  
2. **双轨索引机制 (Dual-Track Indexing)**：  
   * **序号 (Index)**：记录输出表的绝对行位置（1..N），用于监控全量数据规模。  
   * **溯源行号 (Row ID)**：记录该数据在原始索引表中的物理行号，用于执行 1:N 的审计对账，确保原始数据的每一行都已转化为知识。  
3. **递归深度闭环 (Recursive Loop Resolution)**：  
   * 针对 see under 指令，严禁产生逻辑断层。必须自动触发跨条目检索，将被引用目标的身份、地位、核心资助行为直接实例化回填至当前 Row ID。  
4. **零匿名占位原则 (Zero Anonymity)**：  
   * 严禁使用 Portrait X 或 Bust 1。必须遵循 **SENP 5.0 协议**，利用语境（TCR）和外部验证（WACV）构造具备唯一性的描述名称。  
5. **存续状态强标识 (Status Mandatory)**：  
   * 每一个涉及创作或赞助的三元组必须标注：Confirmed (确有其作), Planned (史实计划/未动工), Lost (已毁), Genre (题材特长)。

## **二、 语义处理六大算子 (The Processing Operators)**

### **1\. 触发式原文检索算子 (TCR)**

* **指令**：根据页码定位 CHP-\*.md **正文与脚注**。  
* **任务**：消解指代模糊（如将 "The Cardinal" 还原为 "Francesco Barberini"），捕捉隐藏的中介人。

### **2\. 互联网跨向验证算子 (WACV)**

* **工具**：Getty ULAN/TGN/AAT, Wikipedia, RCT (英国皇室收藏) 目录。  
* **任务**：核实生卒年，标准化地理层级，确认受战争或死亡影响而中断的“未动工”委任。

### **3\. 地理扁平化算子 (Geo-Flattening)**

* **逻辑**：任何涉及物理落位的对象，必须紧随一行该空间的行政隶属关系。  
* **示例**：(Bernini) \-- is\_author\_of \-- (Baldacchino) $\\rightarrow$ (Baldacchino) \-- is\_housed\_at \-- (St Peter's) $\\rightarrow$ (St Peter's) \-- is\_located\_in \-- (Vatican City)。

### **4\. 肖像多维映射算子 (Portrait Mapping)**

* **逻辑**：肖像记录必须拆解为创作轴与社会轴：(作者) \-- is\_author\_of \-- (作品) \+ (作品) \-- depicts \-- (对象) \+ (作者) \-- has\_social\_link \-- (对象)。

### **5\. 中介代理链解构算子 (Broker Chain)**

* **逻辑**：将跨国贸易拆解为代理关系与直接动作：(代理) \-- agent\_for \-- (主顾) \+ (主顾) \-- commissioned\_via \-- (代理) \+ (作品) \-- acquired\_by \-- (主顾)。

### **6\. 题材隔离算子 (Genre Isolation)**

* **逻辑**：将复数名词（Landscapes, Animals）标记为 Genre 状态，并配合 specialized\_in 谓词使用。

## **三、 核心谓词逻辑库 (标准语义边界)**

| 分类 | 精细化谓词 (Predicate) | 语义边界说明 |
| :---- | :---- | :---- |
| **社会/身份** | **spouse\_of** | 仅限正式配偶。 |
|  | **agent\_for** | 财务、艺术或商业采购代理人。 |
|  | **rival\_of** | 明确记录的艺术竞争或政治对抗。 |
|  | **protector\_of** | 政治、宗教或经济层面的核心庇护。 |
| **创作/存续** | **is\_author\_of** | 确证存在的作品（Status: Confirmed）。 |
|  | **planned\_to\_create** | 史实证实有委任但未产生的项目（Status: Planned）。 |
|  | **depicts** | 作品表现的视觉主体。 |
|  | **specialized\_in** | 艺术家的流派特长（如 风景、花卉）。 |
| **空间/地理** | **is\_housed\_at** | 物理现存地（不可移动建筑或现代馆藏）。 |
|  | **is\_located\_in** | 行政层级的严格包含关系。 |
|  | **settled\_in / active\_in** | 定居地（长期）或核心职业活跃地。 |

## **四、 输出格式与审计标准**

### **1\. 结果表格定义 (Markdown/CSV)**

* **序号 (Index)**：全局连续编号。  
* **主体 (Subject)**：标准化实体名。  
* **谓词 (Predicate)**：高精度谓词。  
* **客体 (Object)**：标准化原子实体。  
* **状态 (Status)**：Confirmed / Planned / Lost / Genre。  
* **深度核验逻辑**：详细描述 TCR/WACV 证据链。  
* **Row ID**：原始索引的物理行号锚点。

### **2\. 二次完整性审计逻辑**

* **覆盖律**：原始数据中的每一行 ID 必须在输出结果中出现至少一次。  
* **爆破率**：若作品清单或复合地点未产生多行输出，则视为“解析深度不足”，需重申 V5.3 算子。

## **五、 状态定义补充**

* **Confirmed**：多源证实的实物或事件。  
* **Planned**：史实证明的委任意愿，但由于战争、死亡、拒绝等原因最终未产生实物。  
* **Genre**：指代流派倾向（如“擅长画海景”），非特定单体。
