# **艺术史知识图谱全链路生产规范 (V5.3 \+ V4.9 融合版)**

## **一、 核心算法哲学 (Merged Philosophy)**

本规范的核心在于 **“彻底原子化”** 与 **“全球唯一命名”** 的结合。

1. **1:N 地毯式爆破 (Explosion Strategy)**：  
   * 严禁在 Object 字段中使用复数或逗号。  
   * **地理爆破**：任何地点必须级联。例如：建筑 \-\> 城市 \-\> 国家。  
   * **动作爆破**：将复杂叙述拆解为多个单一动作（如：资助 \+ 委任 \+ 支付）。  
2. **SENP 唯一命名协议 (Naming Protocol)**：  
   * **核心逻辑**：消除一切模糊占位（禁止 "Bust X"）。  
   * **构造算子**：\[Type\]\_\[Subject\]\_\[Artist\]\_\[Date/Context\]。  
   * **应用**：即便作品已丢失或仅在计划中，也必须生成其在图谱中的唯一身份指纹。

## **二、 外部数据查询策略 (Search & Grounding Strategy)**

在处理过程中，利用互联网（Google/Wikipedia/TGN）进行知识对齐是强制性步骤。

### **1\. 触发条件标准 (Trigger Standards)**

必须启动外部查询的情况：

* **地理补全 (Geographic Cascade)**：原文仅提及建筑（如 *Villa Medici*）而未标明城市。  
* **身份固化 (Identity Solidification)**：出现缩写名、职衔（如 *Consul Smith*）或模糊的家族称谓。  
* **SENP 补全**：为了构造唯一 ID，必须确认实体的核心属性（如生卒年、作品创作的具体年份）。  
* **存续状态校验**：当原文对作品现状描述模糊时，需核实其是否已损毁（*Destroyed*）或现藏地（*Current\_Collection*）。

### **2\. 检索源分级**

* **第一级 (TGN/GeoNames)**：用于地理层级校验，确保 is\_located\_in 的准确性。  
* **第二级 (Wikidata/Wikipedia)**：用于获取人物标准名（Subject）和作品元数据。  
* **第三级 (专业美术馆数据库)**：如 *The Royal Collection (RCIN)* 或 *Uffizi*，用于核对全生命周期流转。

## **三、 全生命周期溯源路径 (Data Pipeline)**

每一条艺术品相关的记录必须尝试补全以下链路：  
Commission (委任) \-\> Creation (创作) \-\> Original\_Location (原位) \-\> Brokerage (流转/中介) \-\> Current\_Collection (现藏)

* **中介链解构算子**：A 通过中介 B 买画 C 的逻辑：  
  * (B) \-- agent\_for \-- (A)  
  * (C) \-- acquired\_by \-- (A)  
  * (A) \-- commissioned\_via \-- (B)

## **四、 核心谓词逻辑库 (Standard Predicates)**

| 分类 | 谓词 (Predicate) | 说明 (V4.9 精细化) |
| :---- | :---- | :---- |
| **社会关系** | godfather\_of, executor\_of, debtor\_of | 取代模糊的 link 关系。 |
| **代理关系** | agent\_for, broker\_for, protector\_of | 区分商业代理与政治庇护。 |
| **创作逻辑** | is\_author\_of, planned\_to\_create | 区分确证作品与未竟项目。 |
| **存续真实性** | attributed\_to, workshop\_of, is\_copy\_after | 描述学术鉴定的确定性。 |
| **地理场域** | is\_located\_in, originally\_intended\_for | 描述物理位置与最初的设计意图。 |

## **五、 输出格式与命名规范**

### **1\. 全量结果表字段 (Output Standard)**

* **Index**：绝对序号。  
* **Row ID**：原始索引溯源行号（用于 1:N 审计）。  
* **Subject**：标准人名/实体。  
* **Predicate**：标准谓词。  
* **Object**：遵循 SENP 协议的唯一标识名。  
* **Status**：枚举值：Confirmed, Planned, Lost, Destroyed\_by\_Fire, Unexecuted\_Commission, Genre。  
* **Source**：标注 Page X, Cascade, External Grounding 或 Recursive。

### **2\. 文档命名规则示例**

Haskell\_\[Part\]\_\[Type\]\_\[Ver\]\_\[YYYYMMDD\].\[Ext\]

* *示例*：Haskell\_P\_Atomic\_Extraction\_V4.9\_20260110.md

## **六、 递归解析算法逻辑 (Recursive Resolution)**

1. **See Under 强制解包**：遇到 see under 指令，必须同步检索被引用项。  
2. **属性回写**：将被引用项的社会地位、地理信息回填至当前 Row ID 产生的每一个三元组中。  
3. **时空一致性护栏**：自动核对艺术家创作年份与其生卒年。如果冲突，必须在 Source 中标注“疑误记录”。
