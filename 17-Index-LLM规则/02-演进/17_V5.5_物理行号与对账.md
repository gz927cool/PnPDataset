# **艺术史数据集解析规则规范 (版本 V5.5 \- 终极物理行对账版)**

## **一、 解析核心原则 (Core Principles)**

1. **物理行绝对锚定 (Physical Row Alignment)**  
   * **1:1 映射机制**：原始数据集中的每一行物理文本（包括 Main Entry 为空或只有 Page Numbers 的行）必须视为一个独立锚点。  
   * **Row ID 连续性**：原子表中的 Row ID 列必须严格对应原始 Markdown 表格的行序（从 Row 1 开始）。严禁在处理时跳过任何包含信息的行。  
   * **占位行显式化**：对于无页码的父级引导项或重定向项（如 see under...），必须在 Row ID 中占位，并在“深度核验逻辑”中注明过滤原因。  
2. **语义差异化策略 (V5.4 Semantic Differentiator)**  
   * **逻辑折叠 (Folding)**：针对 active\_in (地理活跃) 或 subject\_of (一般性提及/风格讨论)，将同一主体在同一地点的所有页码合并为一行，页码存入“深度核验逻辑”。  
   * **逻辑爆破 (Exploding)**：针对**具体作品**（画作、雕塑、著作）、**特定社会关系**（赞助、师承、商业伙伴）、**性格/审美特征**或**经济数据**（定价、报酬），必须执行 1:N 拆分，每一项生成独立三元组。  
3. **单文档纯净性 (Document Purity)**  
   * 单次任务输出必须严格限定在当前处理的文档（如 P.md）物理范围内，严禁跨文档携带数据。

## **二、 数据语言与结构标准 (Data Protocol)**

### **1\. 语言规范**

* **三元组数据内容 (S-P-O)**：必须使用 **英语** (English)，以确保谓词的一致性和知识图谱的国际通用性。  
* **元数据说明 (Metadata)**：审计报告、状态描述、深度核验逻辑及备注必须使用 **中文** (Chinese)，以便于深度监测与纠错。

### **2\. 字段定义**

| 字段名称 | 解析要求 | 示例 |
| :---- | :---- | :---- |
| **序号 (No.)** | 全局唯一原子序号，单调递增。 | 101 |
| **主体 (Subject)** | 英语。标准化实体名。补全职衔全名。 | **Pope Innocent X** |
| **谓词 (Predicate)** | 英语。必须使用规范谓词表（见第四章）。 | **has\_identity** |
| **客体 (Object)** | 英语。目标实体、属性值或作品名称。 | **Gianbattista Pamfili** |
| **状态 (Status)** | 中文。描述核验进度。 | **已消解** / **已确认** |
| **深度核验逻辑** | 中文。包含折叠备注、级联依据或作品背景。 | **折叠备注**: p.19-401。 |
| **Row ID** | 原始物理行号。必须与对照数据 1:1 锚定。 | 19 |

## **三、 实体与地理消解规则 (Resolution Rules)**

1. **身份消解 (Identity Resolution)**  
   * **职衔全名化**：通过 has\_identity 将职衔转化为具体人物实体。  
   * **去匿名化**：明确区分同名实体（如西班牙国王 Philip II 与 Philip IV）。  
2. **地理级联 (Geo Cascading)**  
   * **层级补全**：每个地点实体（宫殿、城市）必须通过 is\_located\_in 推断出其上一级行政归属。  
   * *示例*：Prà della Valle | is\_located\_in | Padua \-\> Padua | is\_located\_in | Italy。  
3. **反向互联 (Inverse Linking)**  
   * 如果作品子项中提及赞助人，应回填：Artist | is\_patronized\_by | Patron。

## **四、 V5.5 标准谓词表 (Mandatory Predicates)**

* has\_identity: 身份/全名消解、别名映射。  
* active\_in: 地理活跃记录（执行折叠）。  
* is\_author\_of: 艺术作品、著作、手稿的创作。  
* patron\_of / is\_patronized\_by: 赞助、雇佣、委任关系。  
* has\_social\_link: 社交、学术联系、中介网络、合作关系。  
* is\_located\_in: 地理归属（含建筑、宫殿、城市、国家）。  
* subject\_of: 泛向讨论、风格批评、文献记录。  
* has\_trait: 性格特征、政策取向、审美偏好描述。

## **五、 输出文档结构规范**

1. **文件命名**：{卷标}\_parsed\_results\_\[起始序号\]\_\[结束序号\].md  
2. **文件标题**：\# {卷标}.md 深度解析全量结果表 (规范 V5.5 \- \[阶段描述\])  
3. **内部模块**：  
   * **一、数据完整性审计报告**：对比原始行数、当前原子数，确认 Row ID 状态。  
   * **二、过滤条目及原因说明**：逐条列出 Redirect 或 Placeholder 的物理行及其原因。  
   * **三、全量原子化解析结果表**：标准 Markdown 表格。

## **六、 故障排除清单 (Troubleshooting)**

* 如果行数对不上：检查是否跳过了 Main Entry 为空但有 Sub-entry 的物理行。  
* 如果数据稀释：检查 active\_in 是否执行了 V5.4 折叠逻辑。  
* 如果内容断层：检查地理级联是否漏掉了 Italy 或 Spain 等国家级根节点。
