# **艺术史知识图谱语义抽取规范 (V6.0 生产级核验版)**

本指南确立了将艺术史叙事文献转化为高精度、原子级、全生命周期可溯源知识图谱的终极工程执行标准。它在 V5.3 的基础上引入了“证据链自证”与“客体纯净化”机制，标志着数据从“抓取阶段”进入“生产核验阶段”。

## **一、 核心工程哲学 (The Engineering Core)**

### **1\. 1:N 绝对原子化地毯式爆破 (Absolute Atomicity)**

* **原则**：严禁合并任何语义维度。每一个三元组必须是逻辑上不可再分的最小单元。  
* **属性剥离（二次爆破算子）**：作品的材质、年代、类型、作者必须从原始描述中彻底剥离，并转化为独立的三元组行。  
  * *示例*：原本的 Confirmed\_Oil\_Painting\_X\_1640 必须拆解为：  
    1. (Subject) \-- is\_author\_of \-- (Painting\_X)  
    2. (Painting\_X) \-- created\_in \-- (1640)  
    3. (Painting\_X) \-- has\_material \-- (Oil on Canvas)

### **2\. 纯净化命名协议 (Purified Naming / SENP 2.0)**

* **强制规范**：Subject（主体）与 Object（客体）列**仅允许出现实体的正式名称**。  
* **禁令**：严禁在名称中包含状态（Confirmed\_）、材质（Marble\_）、类型（Oil\_）或作者（by\_X）等描述性前缀。  
* **唯一性**：通过描述性谓词（如 created\_in）建立的逻辑关系来确保实体的唯一性，而非通过 ID 拼接。

### **3\. See Under 深度溯源机制 (Deep Context Retrieval)**

* **触发动作**：遇到索引中的 see under \[目标\] 指令时。  
* **回溯逻辑**：必须利用目标项关联的页码，**实时回溯检索 NotebookLM 中的原书 MD 文档**，阅读具体文字段落。  
* **实质性回填**：严禁在结果中保留跳转指令。必须提取主体在原书语境下的资助、代理或社交细节，填补知识断头。

## **二、 生产流水线 (The Production Pipeline)**

1. **P1: 语义扫描**：识别复合实体、缩进层级、see under 指令。  
2. **P2: 递归解析**：执行跳转，实时回溯章节段落，抓取背景信息回填。  
3. **P3: 触发式联网 (Grounding)**：满足地理孤岛、职衔模糊、材质缺失等阈值时，强制启动 Google/Wiki/TGN 对齐。  
4. **P4: 原子化生成**：执行 1:N 属性剥离，生成物理行关联的 Index。  
5. **P5: 核验自证 (Evidence\_Logic)**：为每一行编写深度核验逻辑。

## **三、 数据表核心字段规范 (Data Schema)**

| 字段名 | 强制定义 | 生产要求 |
| :---- | :---- | :---- |
| **序号 (Index)** | 全局绝对序号 | 记录图谱的物理行数 T。 |
| **溯源行号 (Row ID)** | 原始文献行号 | 用于 1:N 审计对账，精确关联 R 范围。 |
| **主体 (Subject)** | 补全后的标准名 | 必须去匿名化（如 Smith, Joseph 而非 Consul Smith）。 |
| **谓词 (Predicate)** | 标准谓词库 | 严禁模糊谓词，需区分 agent\_for 与 broker\_for。 |
| **客体 (Object)** | **纯净化正式名** | **禁止包含任何属性前缀。** |
| **状态 (Status)** | 存续枚举值 | Confirmed, Planned, Lost, Destroyed\_by\_Fire, Genre。 |
| **依据 (Source)** | 证据来源 | Page X, Cascade, Grounding, Recursive。 |
| **核验逻辑** | **Evidence\_Logic** | **必填。格式：\[算法标签\]: 逻辑描述。** |

## **四、 核验逻辑 (Evidence\_Logic) 标签体系**

必须在核验逻辑列使用以下标准化标签：

* **\[二次爆破\]**：说明如何将年份、材质、作者从原始描述中彻底剥离。  
* **\[地理级联\]**：说明从地标（Villa）到国家（Country）的四级级联证据。  
* **\[递归跳转\]**：记录回溯了哪一页原文，抓取了什么具体语境进行回填。  
* **\[身份固化\]**：说明如何锁定匿名职衔（去匿名化过程）。  
* **\[SENP 剥离\]**：说明如何将原始 ID 转化为纯净化实体名。

## **五、 外部数据强制查询触发机制**

1. **\[地理孤岛\]**：提及建筑或微小地名但未标明城市。  
2. **\[职衔占位\]**：出现 Consul, Marquis, Earl 且名不全。  
3. **\[属性缺失\]**：构建三元组时缺少材质（Marble/Oil）或具体年份。  
4. **\[存续确认\]**：原文提及失踪、火灾或掠夺，需核实最终状态（如 Looted）。

## **六、 审计与闭环报告 (Audit & Reporting)**

每份文档必须在解析结果前包含以下两部分：

1. **数据完整性审计报告 (Integrity Audit)**：统计原始物理行数 (R)、有效原子三元组数 (T) 及审计状态（PASS/FAIL）。  
2. **差异分析 (Gap Analysis)**：记录因 \[Header\]、\[Recursive Resolved\] 或 \[Null Jump\] 被过滤的行号及工程说明。

## **七、 结果文档命名规则**

**强制格式**：Haskell\_\[Letter\]\_Atomic\_V6.0\_R\[Start-End\]\_T\[Total\]\_\[YYYYMMDD\].md

* *示例*：Haskell\_A\_Atomic\_V6.0\_R1-155\_T750\_20260111.md
