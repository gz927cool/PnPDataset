# **艺术史知识图谱语义抽取规范 (V6.0 终极封卷版)**

本指南确立了艺术史文献向高精度知识图谱转化的最高工业级标准。V6.0 终极版在核验版的基础上，强制执行“SENP 2.0 纯净化协议”与“属性绝对原子化剥离”，确保数据在进行大规模图谱融合时具备零歧义性。

## **一、 核心工程哲学 (The Engineering Core)**

### **1\. 1:N 绝对原子化爆破与属性剥离 (Absolute Atomicity)**

* **原则**：严禁合并任何语义维度。材质、年代、作者、类型必须从实体名称中剥离，转化为独立的三元组行。  
* **属性剥离算子**：  
  * 原本记录：Raphael, Deposition, 1507  
  * **终极爆破结果**：  
    1. (Raphael) \-- is\_author\_of \-- (Deposition)  
    2. (Deposition) \-- created\_in \-- (1507)  
    3. (Deposition) \-- has\_material \-- (Oil on Wood)  
    4. (Deposition) \-- is\_located\_in \-- (Galleria Borghese)

### **2\. 纯净化命名协议 (Purified Naming / SENP 2.0)**

* **核心规范**：Subject（主体）与 Object（客体）列**仅允许出现实体的正式官方名称**。  
* **绝对禁令**：严禁在名称中包含状态（Confirmed\_）、材质（Marble\_）、类型（Oil\_）或作者信息（by\_X）。  
* **唯一性维护**：实体的唯一性不再通过 ID 拼接实现，而是通过与其关联的 created\_in、is\_author\_of 等逻辑关系链在图谱空间内精确定位。

### **3\. See Under 深度原文递归回填 (Deep Recursive Retrieval)**

* **机制**：遇到索引中的 see under \[目标\] 指令时，禁止仅标注跳转。  
* **回溯要求**：必须利用目标项页码，**实时回溯检索 NotebookLM 中的原书章节文档**，阅读具体段落语境。  
* **回填生成**：提取该主体在原书真实语境下的资助事实、商业代理或社交细节，填补知识断头。

## **二、 生产流水线标准 (The Production Pipeline)**

1. **语义预扫描**：识别缩进层级、跳转指令及地理关键词。  
2. **递归解包**：执行 see under 跳转，回溯原书章节，抓取背景元数据回填。  
3. **触发式联网 (Grounding)**：满足地理孤岛、匿名职衔、属性缺失等阈值时，强制启动外部知识库对齐。  
4. **原子化生成**：执行属性剥离，生成 1:N 物理行，剥离一切描述性前缀。  
5. **核验自证 (Evidence\_Logic)**：逐行编写逻辑推导证据。

## **三、 数据表字段定义 (Data Schema)**

| 字段名 | 强制规范 | 生产要求 |
| :---- | :---- | :---- |
| **序号 (Index)** | 全局绝对序号 | 记录图谱的最终物理行数 T。 |
| **溯源行号 (Row ID)** | 原始文献行号 | 用于 1:N 审计对账，精确关联 R 范围。 |
| **主体 (Subject)** | 纯净化标准名 | 必须去匿名化（如 Smith, Joseph 而非 Consul Smith）。 |
| **谓词 (Predicate)** | 标准谓词库 | 严禁模糊谓词，区分 agent\_for 与 commissioned\_via。 |
| **客体 (Object)** | **纯净化正式名** | **绝对禁止包含任何属性前缀（如 Confirmed\_, Oil\_ 等）。** |
| **状态 (Status)** | 存续枚举值 | Confirmed, Planned, Lost, Destroyed\_by\_Fire, Genre。 |
| **依据 (Source)** | 证据来源 | Page X, Cascade, Grounding, Recursive。 |
| **核验逻辑** | **Evidence\_Logic** | **必填。格式：\[算法标签\]: 逻辑描述。** |

## **四、 核验逻辑 (Evidence\_Logic) 标签体系**

必须在“核验逻辑”列使用以下标准化标签进行逻辑自证：

* **\[二次爆破\]**：说明如何将年代、材质、作者从描述中彻底剥离并独立成行。  
* **\[地理级联\]**：展示从地标到国家的四级级联逻辑。  
* **\[递归跳转\]**：记录回溯了哪一页原文，抓取了什么具体语境。  
* **\[身份固化\]**：说明匿名职衔的补全过程（去匿名化）。  
* **\[SENP 剥离\]**：说明如何将原始 ID 转化为 100% 洁净的实体名。

## **五、 审计与闭环报告 (Audit & Reporting)**

每份结果文档必须在解析结果前包含以下两部分：

1. **数据完整性审计报告 (Integrity Audit)**：统计原始物理行数 (R)、有效原子三元组数 (T) 及审计状态（PASS/FAIL）。  
2. **差异分析 (Gap Analysis)**：记录因 \[Header\]、\[Recursive Resolved\] 或 \[Null Jump\] 被过滤的行号及工程说明。

## **六、 结果文档命名规则**

**强制格式**：Haskell\_\[Letter\]\_Atomic\_V6.0\_R\[Start-End\]\_T\[Total\]\_\[YYYYMMDD\].md

* *示例*：Haskell\_S\_Atomic\_V6.0\_R1-226\_T845\_20260111.md
